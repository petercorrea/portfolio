Sept 2020

Variables

When writing code, at times we'll want to retain data to access or manipulate later. Imagine writing a app whose sole responsibility was to add two numbers. It'll be nice, to be able to have two "containers" that would hold the value of each number. The first number could be named num1 and the second as num2.

Above, we use the command

let

to

declare

several

variables

and

initialize

their values with 7 and 2. We can then refer to these values in code by their names. In this example we do that to generate their sum. We can also reassign them with different values. Any line with two forward slashes is ignored by the app. We use those lines to write comments. The vertical line of numbers help us keep track of the lines of code.



Types

There are several values we can assign to variables. Values are segregated into

types.

Let's list out JavaScript types and examples of values.

Scope

Variables have certain properties to them that determine where and when it's value can be accessed or changed, we call this

scope.

Scope is important becaue it allows us to seperate access to values or data for several reasons including increasing security, decreasing access or visibility, reducing unindented changes and avoiding naming conflicts with other variables. JavaScript has 5 kinds of scope.

  • 1.

    Global Scope

  • 2.

    Block Scope

  • 3.

    Function Scope

  • 4.

    Nested Scopes

  • 5.

    Lexical Scopes

Any variables not within a pair of curly braces

{}

are in the global scope, their values can be accessed from anywhere within the entire file. But values **within** curly braces are 'scoped' to those braces, only accessible from within the braces. We call that block scope.

console.log()

is an instruction that tells the program to print onto the screen whatever you pass inside the parentheses

()

We can use it to print values we want to evaluate.
